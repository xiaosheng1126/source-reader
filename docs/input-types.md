# Input Types

## 网页类

包括静态网页、JS 渲染网页、博客、官方文档、产品文档和 API reference。

优先目标是获取正文、标题、作者、发布时间、页面结构和关键链接。对于 API reference，要额外提取版本、参数、限制、示例和迁移注意事项。

## 平台类

包括 GitHub、YouTube、B站、X thread、HN、Reddit、V2EX 和公众号。

平台内容要保留上下文：

- GitHub：issue/PR 背景、设计讨论、影响范围、迁移方式。
- 视频：字幕、章节、示例、行动项。
- 讨论串：共识、反对意见、真实经验、噪声和不确定性。
- X thread：观点链路和争议点，不要当作稳定事实。

## 文件类

包括 PDF、论文、本地 Markdown、TXT、HTML、截图和聊天记录。

论文/PDF 要提取：

- 研究问题
- 方法
- 实验或证据
- 结论
- 限制
- 对用户可迁移的实践

截图和聊天记录要标注隐私风险、上下文缺失和置信度。

## 音视频类

包括播客、课程、会议录音和视频。

优先流程：

1. 获取已有字幕或转写文本。
2. 按主题分段。
3. 提取观点、例子、行动项和可沉淀知识。
